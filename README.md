### Commit 规范

commit 规范分成三部分，**type(?scope): subject**， type,subject 为必选，scope 为可选。

#### type 提交类型

- 'feat', // 新功能（feature）
- 'fix', // 修补 bug
- 'docs', // 文档（documentation）
- 'style', // 格式（不影响代码运行的变动）
- 'refactor', // 重构（即不是新增功能，也不是修改 bug 的代码变动）
- 'perf', // 优化性能
- 'test', // 增加测试
- 'revert', // 回滚
- 'chore', // 打包工具等优化
- 'merge', // 合并分支
- 'sync', // 同步其它分支的 bug 修复等

#### scope 影响的范围

commit 影响的范围和模块，可以不写

#### subject 提交主题

本次提交修改的描述，说明修改了什么 bug，增加了什么新功能等。
